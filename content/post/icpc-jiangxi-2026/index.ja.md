## A — 導入問題

**题意**：2つの数 m, n が与えられ、m mod n と n mod m を計算する。

**解法**：m % n と n % m をそのまま出力する。入力の順序と形式に注意。

導入問題だが、問題文にやや紛らわしい部分があるので注意深く読むこと。

計算量 O(1)。

## B — 性質問題

**题意**：ある特殊な性質を持つ数列が与えられる。

**解法**：この問題の鍵は性質の観察にある。データ範囲が小さいため、O(n²) の全探索で十分。

一部のチームがここで詰まっていたが、実際には全ての可能なケースを直接全列挙すれば解ける。

計算量 O(n²)。

## C — 行列累乗最短経路

**题意**：グラフが与えられ、各辺に通過時間がある。時間は行列の形で格納される。複数の更新操作があり、各更新後に時間に関する何らかの最値を求める。

**解法**：時間情報を行列に格納し、行列積の加算を min、乗算を max に置き換える。

これにより行列累乗がFloydの高速化バージョンとなる。更新のたびに行列の冪乗を再計算する。

計算量 O(n³ log T)（更新1回あたり）。

## D — 全方位木DP

**题意**：木の各ノードに重みがある。各ノードを根としたときの、ある重みに関する統計量を求める。

**解法**：古典的な全方位木DP（rerooting DP）。

1. 1回目のDFS：根（例えば1）を固定し、部分木の情報（サイズ、重み和など）を前処理
2. 2回目のDFS：根から子ノードへ遷移し、親ノードの答えを使って子ノードの答えを計算

遷移式は統計量によって異なるが、核心は `ans[v] = ans[u] - 寄与(vの部分木) + 寄与(残りの部分)`。

計算量 O(n)。

## F — 転送ポイント

**题意**：グラフ中に複数組の転送ポイントがある。1組の転送ポイントは1つのノードに縮約できる。転送ポイントに接続する辺の本数の差は2以下で、差が0より大きいのは端点のみ。何らかの最適な方案を求める。

**解法**：
- 各組の転送ポイントを1つのノードに縮約
- 転送ポイントに接続する辺の本数の差は ≤ 2
- ネットワークフローまたはオイラー経路の構築で求解

具体的には、問題をグラフ理論における何らかのマッチングまたは経路被覆問題に変換する。

## G — セグメント木 + 文字列

**题意**：文字列マッチング問題で、不一致ポインタの前綴・後綴情報を維持する必要がある。

**解法**：セグメント木を用いて区間内の不一致前綴・後綴情報を管理する。

重要な観察：不一致ポインタ（border）の前綴と後綴の情報はマージ可能。セグメント木の各ノードは以下を保持：
- 区間の文字列の前綴不一致情報
- 区間の文字列の後綴不一致情報

クエリ時には左右の区間情報をマージする。

計算量 O((n+q) log n)。

## H — Nim ゲーム

**题意**：Nim ゲームの変種。2進数の最下位ビットのパリティに関係する。

**解法**：2進数表現を分析する。

x = a xor b とする（a, b はある2つの数）。x の2進数での最下位1のビット位置を調べる：

- x が奇数の場合（最下位ビットが1）、先手必勝
- それ以外の場合、x/2 について再帰的に考える

これは本質的にNimゲームの一種の変種で、SG関数は再帰的に構築できる。

## K — BSGS

**题意**：離散対数問題を解く。a^x ≡ b (mod p) を満たす最小の x を求める。

**解法**：Baby Step Giant Step アルゴリズム（BSGS）。

x を x = i·m - j と表す（ただし m = ceil(sqrt(p))）。a^j のハッシュテーブルを前処理し、i を列挙して照合する。

境界ケースと解なしの場合に注意。

計算量 O(sqrt(p))。

## L — 数論 +  Moのアルゴリズム

**题意**：φ(xy) = φ(x)φ(y)gcd(x,y) / φ(gcd(x,y)) の公式に関連し、区間クエリを行う。

**解法**：
オイラーのφ関数の積の公式を利用：

$$φ(xy) = φ(x)φ(y)\frac{gcd(x,y)}{φ(gcd(x,y))}$$

区間クエリでは gcd 情報の管理が必要。Moのアルゴリズムでクエリをオフライン処理し、各数の因子情報を併用する。

計算量 O((n+q) sqrt(n))。
