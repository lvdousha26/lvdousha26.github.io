## 为什么需要方法论

新手读论文常犯的错：从第一页逐字啃到最后，花了 4 小时读完一篇，过两天全忘了。正确的做法是**分层过滤**——大部分论文只需要读 10% 的内容就能判断是否值得精读。

<!--more-->

## 三遍法（Three-Pass Approach）

来自 S. Keshav 的经典论文阅读方法，一篇论文最多读三遍。

### 第一遍：快速扫描（5-10 分钟）

**只看：标题、摘要、引言第一段、结论、图表**

目标回答 5 个问题：

1. **类型**：是理论分析、系统实现还是实验评估？
2. **背景**：和哪些论文相关？理论依据是什么？
3. **正确性**：假设成立吗？
4. **贡献**：主要创新点是什么？
5. **清晰度**：写得清楚吗？

这遍之后决定：**不读了 / 再读第二遍 / 先放一放**。大部分论文在这一关就被筛掉了。

### 第二遍：仔细阅读（1 小时）

通读全文但不死磕细节，标注关键术语和未理解的证明步骤。

- 浏览方法部分但不深究推导
- 在页边记下关键点
- 注意图表中没在第一遍注意到的细节
- 标记没看懂的参考文献，之后追读

读完应能向别人**口头总结论文**：它解决了什么问题，用了什么方法，结果如何。

### 第三遍：深入重现（4-5 小时，仅顶级论文）

- 从头到尾复现推导过程
- 设想如果自己做这个研究会怎么做
- 找出隐含假设和潜在问题
- 思考如何改进或扩展

读完应能重构论文的结构和论证，找出强项和弱点。

## 具体工作流

### 选题与搜索

```bash
# 工具推荐
Semantic Scholar  # AI 辅助搜论文，显示被引量和影响力
Google Scholar     # 经典方案
DBLP              # CS 领域最全
Connected Papers  # 可视化论文引用关系图
arXiv Sanity      # 按关键词订阅最新预印本
```

**搜索策略：**
1. 找 3-5 篇近两年高引综述（survey/overview），建立领域全景
2. 从综述的引用链中找里程碑论文（被引 500+ 的根节点）
3. 找最新的顶会论文了解 SOTA
4. 借助 Connected Papers 发现遗漏的关键工作

### 筛选优先级

| 来源 | 可信度 | 适合阶段 |
|------|--------|----------|
| 顶会/顶刊（CVPR/NeurIPS/TPAMI 等） | 高 | 所有阶段 |
| arXiv 预印本 | 中 | 追最新方向 |
| 知名团队主页 | 高 | 找方向 |
| 工业界技术报告 | 中 | 工程实践 |
| 中文综述 | 中 | 入门 |

判断一个会议/期刊的级别：查 [CCF 推荐列表](https://www.ccf.org.cn/Academic_Evaluation/By_category/) 或 `csrankings.org`。

## 做笔记：别信任自己的记忆力

### 推荐工具

| 工具 | 适用场景 |
|------|----------|
| **Notion** | 结构化笔记、数据库视图 |
| **Obsidian / Logseq** | 双向链接、知识图谱 |
| **Zotero** | 文献管理 + PDF 批注 |
| **Marginnote** | 深度 PDF 阅读 + 脑图 |
| **Markdown + Git** | 纯文本，永久可控 |

### 笔记模板

每篇论文一个页面：

```markdown
# [论文标题]

## 基本信息
- 作者/机构：
- 发表：CVPR 2024
- 链接：
- 代码：
- 标签：

## 一句话总结
用一句话说清楚这篇论文做了什么。

## 核心问题
要解决什么问题？为什么现有方法不够？

## 方法与创新点
1.
2.
3.

## 实验与结论
- 数据集：
- 对比方法：
- 核心指标：
- 关键消融发现：

## 我的思考
- 优点：
- 不足：
- 可借鉴的 idea：
- 和 xxx 论文的关系：

## 下一步
- [ ] 读参考文献 xxx
- [ ] 看代码复现
```

坚持一个笔记模板比反复寻找完美格式更有价值。

## 批量阅读进阶：构建知识地图

读够 30-50 篇后开始整理：

```
综述 (3-5 篇)
  ├── 子方向 A (10-15 篇)
  │     ├── 里程碑论文 → 改进论文 → SOTA
  │     └── 数据集 / Benchmark
  ├── 子方向 B (10-15 篇)
  │     └── ...
  └── 待探索 (标记潜在的 5 篇)
```

用 Obsidian 或 Logseq 的双向链接功能，论文之间互相关联，形成知识网络而不是碎片记忆。

## 常见误区

| 错误做法 | 正确做法 |
|----------|----------|
| 每篇都从头啃到尾 | 三遍过滤，90% 只到第二遍 |
| 只读最新 arxiv | 先读里程碑论文，建立根基 |
| 不做笔记 | 当天读完当天记，每周回顾 |
| 英文恐惧 | 先读中文综述，再用翻译辅助英文 |
| 囤积 PDF 不读 | 每周固定 3 小时阅读时间 |
| 跳过实验部分 | 实验设计是方法论的重要环节 |

## 节奏建议

| 阶段 | 阅读量 | 重点 |
|------|--------|------|
| 研一上 | 3-5 篇/周 | building knowledge map |
| 研一下 | 5-8 篇/周 | 深入子方向，精读 SOTA |
| 研二上 | 5 篇/周精选 + 代码阅读 | 自己的方向收窄 |
| 写论文 | 10+ 篇/周速读 related work | 快速扫相关方法 |

质量 > 数量。一篇精读胜过十篇扫读。但要精读到点子上——**和你的研究直接相关的才精读**。
