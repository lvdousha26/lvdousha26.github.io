Claude CodeはAnthropicが提供するコマンドラインAIプログラミングツールです。ターミナル上で直接コードを読ませたり、バグを修正させたり、機能を実装させたりできます。Copilotのような補完機能とは違い、真の対話型コラボレーションを実現します。

<!--more-->

## インストール

```bash
# npmでグローバルインストール（推奨）
npm install -g @anthropic-ai/claude-code

# 確認
claude --version
```

初回使用時はAnthropic API KeyまたはClaudeのサブスクリプションが必要です：

```bash
# 方法1：API Key（従量課金）
export ANTHROPIC_API_KEY="sk-ant-..."

# 方法2：Claudeサブスクリプションアカウントでログイン
claude login
```

## 基本的な使い方

```bash
# 直接質問
claude "このプロジェクトのディレクトリ構造を説明して"

# インタラクティブモード（最もよく使う）
claude

# ファイルを指定
claude "src/main.py にどんな潜在的問題がある？"
```

インタラクティブモードに入ると、チャットのように要件を伝えられます：

```
> このプロジェクトにログミドルウェアを追加して
> READMEにインストール手順が抜けてるから補足して
> この関数が長すぎるから2つに分割して
```

Claude Codeは自動的にコードを読み、修正を行い、テストを実行し、満足のいく結果が得られるまで反復します。

## プロジェクトコンテキスト — CLAUDE.md

Claude Codeは心を読みません。プロジェクトのルートに `CLAUDE.md` を置いて、プロジェクトのルール、コードスタイル、注意事項を伝えましょう：

```markdown
# CLAUDE.md

## プロジェクト
FastAPIバックエンド、Python 3.11、Poetryで依存関係管理

## ルール
- コードを読んでから修正すること。推測しない
- 必要な場合を除き、新しい依存関係を追加しない
- 関数シグネチャを変更するときは、すべての呼び出し箇所を同期して更新する
- キー、トークンなどをハードコードしない

## テスト
pytest、コード修正後は pytest -x を実行
```

このファイルはClaude Codeが毎回自動的に読み込みます。プロジェクトの説明書を渡すようなものです。

## 権限制御

デフォルトでは、Claude Codeがシェルコマンドを実行するたびに確認が必要です。設定で操作タイプごとにホワイトリストを登録できます：

```bash
# 現在の権限設定を確認
claude config

# ファイル読み取りを許可（確認不要）
claude config allow read
```

または `~/.claude/settings.json` に直接記述：

```json
{
  "permissions": {
    "allow": [
      "Bash(git diff:*)",
      "Bash(git log:*)",
      "Bash(npm test:*)"
    ]
  }
}
```

## コアワークフロー

```
1. 要件を説明
      ↓
2. Claude Codeがコードを読み、ドキュメントを確認
      ↓
3. 修正案を提示
      ↓
4. あなたが確認 → 自動でコード修正
      ↓
5. テスト実行 → 問題修正 → 完了
```

実際の効果：バグの現象を説明するだけで、原因の特定から修正、検証までを一貫して行ってくれます。

## 便利なテクニック

### 複数ターンの会話で段階的に

いきなり大きな要件を伝えないでください。まずは現状を理解させましょう：

```
> src/auth.py を読んで、認証フローを説明して
> 今からログイン失敗回数の制限を追加したい。5回失敗したら30分ロック
> Redisをカウンターとして使って、キーには有効期限を設定して
```

### @ でファイルを参照

```
> @src/models.py @src/routes.py この2つのファイル間で循環インポートはある？
> @.github/workflows/ CIのNode 16を20にアップグレードして
```

### 自分でドキュメントを調べさせる

Claude Codeは自身の限界を認識しており、自発的に「このAPIの変更点が確かではないので調べます」と言ってくれます：

```
> Prisma 5.xの新しい構文でデータベースクエリを書き直して
```

### コードレビュー

```bash
# 現在のブランチの変更をレビュー
claude "git diff main...HEAD の変更をレビューして"
```

### commit messageを書く

```bash
git diff | claude "conventional commit message を書いて"
```

## よく使うコマンド一覧

| 操作 | コマンド |
|------|----------|
| インタラクティブモード | `claude` |
| 単発の質問 | `claude "質問"` |
| 前回のセッションを継続 | `claude --continue` |
| セッション一覧 | `claude sessions` |
| モデル切り替え | `/model opus` / `/model sonnet` / `/model haiku` |
| ヘルプ表示 | `/help` |
| コンテキストクリア | `/clear` |
| 終了 | `/exit` または `Ctrl+C` |

## Copilot / Cursorとの違い

| | Claude Code | Copilot | Cursor |
|------|-------------|---------|--------|
| 実行形態 | CLIターミナル | IDEプラグイン | IDE（VS Codeフォーク） |
| 対話方式 | 対話型 | 補完＋インラインチャット | 補完＋サイドバーチャット |
| コード変更範囲 | プロジェクト全体 | 主に現在のファイル | プロジェクト全体 |
| 自動実行 | テスト実行、依存関係インストール可 | 不可 | 一部対応 |
| 価格 | API従量課金 / サブスクリプション $20/月 | $10/月 | $20/月 |

Copilotは行単位の補完に優れ、CursorはGUIでの操作に向いています。Claude Codeの強みはターミナルネイティブであること、複数ファイルのリファクタリング、自動化ワークフローにあります。

## 初心者へのアドバイス

1. **最初にCLAUDE.mdを書く** — これがAIへのプロジェクト説明書です
2. **小さなタスクから始める** — まずはtypo修正やコメント追加から始めて、徐々に信頼を築く
3. **盲信しない** — 変更内容を必ず確認してからコミットする
4. **@参照を活用する** — 曖昧な説明より正確なコンテキストを与える
5. **複雑な要件は段階的に** — 1回で大きなタスクを頼むより、小さなステップに分割する

コンテキストを適切に管理すれば、Claude Codeは膨大な繰り返し作業を省いてくれます。使えば使うほど、あなたのプロジェクトに詳しくなっていきます。
